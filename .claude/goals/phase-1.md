# Goal: Phase 1 — Cardano Flutter SDK Core

Build Phase 1 of the production-grade Cardano Flutter SDK with complete end-to-end FFI integration, test coverage, and documentation.

## Success Criteria

**The following must ship together:**

1. **Toolchain bootstrap**
   - `flutter_rust_bridge` v2.x codegen pipeline working end-to-end
   - Rust stable edition 2021 with `cardano_flutter_rs` crate
   - All lints passing: `cargo clippy --all-targets -- -D warnings` and `flutter analyze`
   - Build scripts for all platforms (iOS, Android, web, macOS, Linux, Windows) verified on at least two targets

2. **CSL wrapper in Rust**
   - `cardano-serialization-lib` v15+ integrated and compiling
   - Error types defined with `thiserror`, mapped to Dart exceptions
   - No panics in public API surface
   - At least 3 proof-of-concept functions exposed (e.g., address validation, serialization round-trip, CDDL inspection)

3. **Dart API surface (Phase 1 read-only)**
   - Generated bindings in `dart/lib/src/bridge_generated.dart` (never hand-edited)
   - Public APIs with dartdoc comments, usage examples
   - Type-safe wrappers for CSL types
   - `Future`-based async boundary (Rust is sync for Phase 1)

4. **Example app**
   - Demonstrates at least one end-to-end flow (e.g., load an address, validate it, inspect its structure)
   - Runs on iOS simulator and Android emulator
   - No hardcoded secrets; config externalized to `.env.example`

5. **Tests**
   - Unit tests for all Rust public functions (via `cargo test`)
   - Dart unit tests for all public bindings (via `flutter test`)
   - Integration test against Cardano testnet preview (address lookup, UTXO query)
   - Fixtures in `tests/fixtures/` with test mnemonics

6. **Documentation**
   - README updated with setup instructions, toolchain prereqs, build/test commands
   - Architecture diagram (Mermaid or PNG) showing Dart → FFI → Rust → CSL flow
   - Inline code comments for non-obvious invariants only (not "what" — that's in names)

## Agent Workstreams

Deploy these in parallel. Coordinator reviews outcomes and gates ship.

### 1. Research Agent
- **Task:** Deep-dive on `flutter_rust_bridge` v2.x best practices, CSL v15 API surface, and platform-specific FFI quirks
- **Deliverable:** research summary doc with: setup checklist, CSL function candidates for Phase 1, known gotchas per platform, example FFI bindings from other projects
- **Owner:** assigned
- **Blocked by:** none

### 2. Rust Scaffolding Agent
- **Task:** Bootstrap `rust/cardano_flutter_rs`, wire CSL dependency, implement error types, create 3 PoC functions (address validation, serialization, CDDL inspection)
- **Deliverable:** compiling Rust crate with `cargo test` and `cargo clippy` passing; no panics in public API
- **Owner:** assigned
- **Blocked by:** Research Agent (CSL API surface and function choices)

### 3. Dart Bindings & Codegen Agent
- **Task:** Set up `flutter_rust_bridge` codegen pipeline, generate initial bindings from Rust PoC functions, create type-safe Dart wrappers with dartdoc examples
- **Deliverable:** `bridge_generated.dart` autogenerated; public Dart APIs with examples; `flutter analyze` passing
- **Owner:** assigned
- **Blocked by:** Rust Scaffolding Agent (Rust code to bind to)

### 4. Test Automation Agent
- **Task:** Write Rust unit tests for all public functions; Dart unit tests for bindings; integration test harness for testnet preview
- **Deliverable:** `cargo test` and `flutter test` both passing; integration test executable against public Cardano preview testnet; fixtures in `tests/fixtures/`
- **Owner:** assigned
- **Blocked by:** Rust Scaffolding + Dart Bindings Agents

### 5. Example & Docs Agent
- **Task:** Build example app demonstrating one end-to-end flow (e.g., "Enter an address → SDK validates it → show its structure"); update README with setup, build, test commands; create architecture diagram
- **Deliverable:** example app building and running on iOS simulator + Android emulator; updated README; architecture diagram
- **Owner:** assigned
- **Blocked by:** Dart Bindings Agent + Test Automation Agent (to reference in docs)

## Scope Guards

- **Do not** reimplement Cardano crypto in Dart. Use CSL via FFI.
- **Do not** ship Phase 2 features (signing, settlement, group management) in Phase 1.
- **Do not** hand-edit `bridge_generated.dart`.
- **Do not** panic in Rust public API. Use error types.
- **Do not** skip testnet integration tests before CSL version bumps.
