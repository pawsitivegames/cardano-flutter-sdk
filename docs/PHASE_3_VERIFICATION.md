## Known Limitations

1. ~~**Plutus cost models**: empty `Costmdls`~~ тАФ **Fixed (2026-05-26)**: `build_script_tx`
   now uses hardcoded Conway V1/V2/V3 cost models sourced directly from the CSL v15 source
   (`TxBuilderConstants::plutus_conway_cost_models`, which is `pub(crate)`). The
   `script_data_hash` is computed with correct cost models; Plutus txs will pass node
   validation.

2. **Ex-units are user-supplied**: `PlutusInput.ex_units_mem/steps` are passed in
   directly. Production use requires off-chain script evaluation (e.g. via Ogmios
   or Cardano CLI).

3. **Reference inputs**: accepted and passed to CSL builder, but full CIP-31
   reference-script spending is not wired.
