## Known Limitations (Phase 3.5)

1. **Plutus cost models**: `build_script_tx` uses `Costmdls::new()` (empty) because
   `TxBuilderConstants::plutus_vasil_cost_models()` is `pub(crate)` in CSL v15 and
   cannot be called from external crates. The `script_data_hash` in the tx body is
   computed with empty cost models, which will be rejected by node validation.
   **Fix**: switch backend to CML (which exposes cost models), or extract cost models
   from protocol parameters (Blockfrost returns them as base64 JSON).

2. **Ex-units are user-supplied**: `PlutusInput.ex_units_mem/steps` are passed in
   directly. Production use requires off-chain script evaluation (e.g. via Ogmios
   or Cardano CLI).

3. **Reference inputs**: accepted and passed to CSL builder, but full CIP-31
   reference-script spending is not wired.
